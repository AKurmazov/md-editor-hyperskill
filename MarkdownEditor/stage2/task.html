Description

Before you will start implementing this project, we need to think about what functionality will be offered to a user. Remember that tutorial from the previous stage? Well, let's go through it one more time and recall some basic formatters:

  • plain — just a normal text without any formatting
  • bold/italic — self-explanatory
  • inline-code — for example,  python editor.py
  • link — for example, a link to google
  • header — look at the Description header above
  • unordered-list — you are looking at it right now
  • ordered-list — the same list but with enumerated elements
  • new-line — switches to the next line

I think you have already guessed that you will have to implement these formatters in your editor. Also, it would be cool to add some special commands to our tool. For instance,

  • !help — prints all the available formatters
  • !done — saves the markdown and exits

OK, let's get into coding!

Objectives

So, we know the functionality of the editor, let's share it with the users! 

In this stage, I ask you to implement a helper function (for example, help) that will print all the available formatters (those that we've discussed above) and special commands. Also, you have to do an exit function (without any results' storage for the time being). Let's call it done

Next, let's continuously prompt a user for an input

- Choose a formatter:

If the input is

  • !help — the helper function help is called
  • !done — the exit function done is called

Else if the input is one of the formatters (plain, bold, italic, etc.) — the user is prompted for an input once again

Otherwise — the following error is printed: Unknown formatting type or command

Examples

A working session with the tool might look like this after completing this stage:

	$ python markdown-editor.py
- Choose a formatter: > non-existing-formatter
Unknown formatter or command. Please try again
- Choose a formatter: > !help
Available formatters: plain bold italic link inline-code ordered-list unordered-list new-line
Special commands: !help !done
- Choose a formatter: > header
- Choose a formatter: > ordered-list
- Choose a formatter: > !done
The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
