Description

Congratulations! You've made it to the most interesting part of the project — you will now start implementing the formatters that we have discussed before. For your convenience, I will paste the list from the previous stage here:

  • plain
  • bold/italic
  • inline-code
  • link
  • header
  • new-line

We won't consider ordered and unordered lists now, but you'll get to them in the next stage!

Objectives

You have to make your program recognize the above-mentioned formatters. I suggest implementing a separate function for each of them — it is a good practice to keep your code structured, and it will be easy to find a bug if something doesn't work as intended. 

It should work the following way:

  1. A user types in a formatter
  2. If the formatter doesn't exist, the editor should print the corresponding error: Unknown formatter or command. Please try again
  3. The user is asked to type some input requested by the formatter: > Text: <user's input> 
  4. Your program saves the text with the chosen formatter applied to it and prints the whole markdown

Notice that different formatters may ask a user to type different input.

For new-line, nothing is required to input

For plain, bold, italic, and inline-code, only text is required

	-- Text: > Some text

For header, a level and text are required. Don't forget to check that the level is within the range from 1 to 6. If it violates this rule, print the corresponding error: Level should be within the range from 1 to 6

	-- Level: > 4
-- Text: > Hello World!

For link, a label and a URL are required

	-- Label: > Tutorial
-- URL: > https://www.markdownguide.org/basic-syntax/
